Jira Planning Tool — quick guide

This browser extension helps align team capacity across periods and a backlog with issues tied to plan periods or quarters. Data is stored locally in your browser profile (Chrome / Edge).

Opening the app

Open the extension from the toolbar icon and choose Open Jira Planning Tool (or the equivalent action from the extension description). A separate tab opens with the planning UI.

Plans

  • Create Plan — create a plan: name, start period (quarter and year), default working days, estimation type (Story Points or Man-days), resource grouping, and more.
  • The dropdown next to the button switches between saved plans. Each plan has its own settings and tables.
  • If the plan has no periods or capacity rows yet, an overlay may prompt you — use its buttons (e.g. create plan or period).

Capacity tab

People, roles, and available capacity per period (columns with dates / quarters).

  • Rows are team members: name, role, load %, working days, days off, estimation per day, and derived fields (available capacity, planned, balance).
  • The + button near the table adds a capacity row; the + on the right adds period (quarter) columns.
  • View: Full — full grid; Compact — simplified (some per-member and days-off columns hidden); in compact mode the table width follows content instead of stretching.
  • By team / By roles is set in Settings: one team summary or split by role (both Capacity and Backlog).

Backlog for planning tab

  • Issue list (key, summary, status, priority, type, estimate). The Period column selects which plan period counts demand toward; without a period the issue does not contribute to “planned” on Capacity.
  • Import backlog from Jira loads issues via JQL (server base URL and the correct estimate field — see below).
  • First column: checkboxes — select one or many rows; the header selects all. Delete selected removes chosen rows after confirmation.
  • Re-import updates existing issues by key and appends new ones. If you deleted an issue locally but it appears in the JQL result again, it is added back.

Settings (gear icon)

  • Estimation type — Story Points or Man-days for metrics and backlog.
  • For Story Points — how “team Story Points per day” is derived (average across members or a fixed team value).
  • Working days for all rows — default working days for capacity rows; new rows pick this up from Settings.
  • Load (%) for all rows — default load for every row when you save Settings.
  • Resource grouping type — by team or by roles; in by-roles mode you can manage the plan’s role list.

Jira import

In the import dialog, provide:

  • Your Jira base URL (how you open the instance in the browser). The extension uses the Search API; access must match your logged-in account.
  • JQL — which issues to load (same idea as Jira Issue Navigator).
  • Estimate field depends on plan type: Story Points usually need a custom field id (e.g. customfield_…); Man-days use a time/estimate field (often timeoriginalestimate or similar — hints are in the form).

Import does not replace the whole backlog: it merges on the normalized issue key (e.g. PROJECT-123).

Where data lives

Plans and tables are stored in the browser’s local storage (chrome.storage.local). Clearing site data or removing the extension may delete plans — copy important values elsewhere if needed (a dedicated file export may not be available in this version).

Jira Planning Tool — краткая инструкция

Расширение для браузера помогает согласовать ёмкость команды по периодам и бэклог с привязкой задач к кварталам или периодам плана. Данные хранятся локально в профиле браузера (Chrome / Edge).

Как открыть приложение

Откройте расширение через иконку на панели инструментов и выберите действие Open Jira Planning Tool (или аналогичное название из описания расширения). Откроется отдельная вкладка с интерфейсом планирования.

Планы

  • Create Plan — создать новый план: имя, стартовый период (квартал и год), рабочие дни по умолчанию, тип оценки (Story Points или Man-days), режим группировки ресурсов и др.
  • Выпадающий список рядом с кнопкой создания — переключение между сохранёнными планами. У каждого плана свои настройки и таблицы.
  • Если в плане ещё нет периодов или строк ёмкости, часть экрана может быть перекрыта подсказкой — используйте кнопки на оверлее (например, создать план или период).

Вкладка «Capacity»

Здесь задаются люди, роли и доступная ёмкость по периодам (колонки с датами / кварталами).

  • Строки — участники: имя, роль, нагрузка в процентах, рабочие дни, выходные, оценка в день и производные поля (доступная ёмкость, план, баланс).
  • Кнопка + у таблицы добавляет строку ёмкости; отдельная + справа — добавление колонок периода (квартала).
  • View: Full — полная таблица; Compact — упрощённый вид (скрыты часть колонок «на человека» и выходные); ширина таблицы в компактном режиме подстраивается под содержимое.
  • Режим By team / By roles задаётся в настройках: сводка по команде или разбивка по ролям (и в Capacity, и в Backlog).

Вкладка «Backlog for planning»

  • Список задач (ключ, summary, статус, приоритет, тип, оценка). Колонка Period — в какой период плана относится спрос по задаче; без выбора периода задача не участвует в расчёте «запланировано» на Capacity.
  • Import backlog from Jira — загрузка задач из Jira по JQL (нужен базовый URL сервера и корректное поле оценки — см. ниже).
  • Первая колонка — чекбоксы: можно выбрать одну или несколько строк, в шапке — «выбрать все». Кнопка Delete selected удаляет выбранные строки из бэклога после подтверждения.
  • Повторный импорт обновляет уже существующие задачи по ключу и добавляет новые. Если задачу удалили локально, но она снова попала в результат JQL, она появится в таблице снова.

Настройки (иконка шестерёнки)

  • Estimation type — Story Points или Man-days для метрик и бэклога.
  • Для Story Points — режим расчёта «Story Points в день» для команды (среднее по людям или фиксированное значение).
  • Working days for all rows — базовое число рабочих дней для строк ёмкости; для новых строк оно подставляется из этого значения.
  • Load (%) for all rows — процент загрузки по умолчанию для всех строк при сохранении настроек.
  • Resource grouping type — «по команде» или «по ролям»; при режиме по ролям доступно управление списком ролей плана.

Импорт из Jira

В диалоге импорта укажите:

  • Базовый URL вашего Jira (как вы открываете инстанс в браузере). Расширение использует API поиска; доступ и права должны соответствовать вашей учётной записи в браузере.
  • JQL — какие задачи загрузить (как в Jira Issue Navigator).
  • Поле оценки зависит от типа оценки плана: для Story Points обычно нужен id кастомного поля (например, customfield_…); для Man-days — поле времени/оценки (часто timeoriginalestimate или аналог; подсказки есть в форме).

Импорт не заменяет весь бэклог целиком: выполняется слияние по нормализованному ключу задачи (например, PROJECT-123).

Где хранятся данные

Планы и таблицы сохраняются в локальном хранилище браузера (chrome.storage.local). Очистка данных сайта или удаление расширения может привести к потере планов — при необходимости заранее экспортируйте или зафиксируйте важные значения вне расширения (отдельного экспорта в файле в текущей версии может не быть).

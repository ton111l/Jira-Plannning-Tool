1. Ручное тестирование с Vite + @crxjs/vite-plugin (HMR)
Это «интерактивная подгрузка»: dev-сервер собирает расширение в dist/ и подмешивает изменения без цикла «собрал → переустановил» вручную каждый раз.

Первичная настройка (один раз на машине)
- Установите Node.js LTS (вместе с npm): https://nodejs.org/
- В корне репозитория выполните: npm install
- Папка dist/ в .gitignore — её создаст dev или build; до первого запуска папки может не быть.

Запуск dev с HMR
- В корне проекта: npm run dev   (эквивалент: npm run dev:extension)
- Дождитесь сообщения Vite, что сервер поднят (порт по умолчанию 5173). Оставьте терминал открытым.

Подключение расширения в браузере
- Chrome: chrome://extensions  |  Edge: edge://extensions
- Включите «Режим разработчика» / Developer mode
- «Загрузить распакованное» / Load unpacked → укажите каталог:
  ...\Jira-Plannning-Tool\dist
  Важно: выбирать именно папку dist, а не корень репозитория.
- При ошибках «файл занят» на Windows (OneDrive/антивирус) — закройте лишние вкладки с расширением, при необходимости перезапустите dev.

Как проверять
- Иконка расширения: по манифесту открывается вкладка с приложением (src/app.html через background)
- Content scripts: откройте страницу Jira — скрипты из манифеста подхватятся
- Если правки не подхватились: на chrome://extensions нажмите «Обновить» (Reload) у этого расширения один раз; дальше CRXJS обычно обновляет без полной переустановки

Итог: npm run dev + Load unpacked → dist + нужный сайт для интеграционной проверки.

2. Автотесты в проекте — не через npm run dev по умолчанию
Что	Как запускать
Smoke (Node)	node tests/smoke.test.mjs — логика модулей, без браузера и без Vite.
E2E (Playwright)	В tests/e2e.browser.mjs захардкожен URL http://127.0.0.1:4173/src/app.html — это сценарий под статический сервер на 4173, т.е. скрипт npm run dev:legacy (browser-sync), а не стандартный порт Vite (обычно 5173).
Типичный порядок для E2E:

# терминал 1
npm run dev:legacy
# терминал 2
npx playwright install chromium   # один раз, если ещё не ставили
node tests/e2e.browser.mjs
Если хотите гонять E2E именно на Vite dev, нужно поднять Vite на том же порту/URL, что и в тесте, или поменять URL в e2e.browser.mjs — сейчас проект это явно не связывает.

3. Сборка без dev-сервера
npm run build
Проверка уже упакованного расширения вручную — снова Load unpacked → папка dist после npm run build.

Кратко: для интерактивной разработки и проверки расширения используйте npm run dev + загрузка dist в браузере; для автоматического E2E из репозитория — dev:legacy на 4173 и node tests/e2e.browser.mjs.
